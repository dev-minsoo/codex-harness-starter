# Codex Harness Starter

이 저장소는 사용자가 빈 디렉토리에서 출발해 실제 작업에 들어가기 전에 목적에 맞는 하네스를 만들 수 있도록 돕는 Codex 플러그인입니다.

## 핵심 산출물

`harness-creator` 스킬을 사용할 때는 메타 문서보다 즉시 실행 가능한 결과를 우선합니다.

가장 먼저 유용해야 하는 출력은 다음입니다.

- 어떤 하네스를 만들지
- 역할을 몇 개로 둘지
- 첫 `SKILL.md`에 무엇을 쓸지
- 다음에 바로 실행할 첫 프롬프트가 무엇인지

## 기본 동작

사용자의 목표를 지원할 수 있는 가장 작은 하네스를 우선합니다.

역할 분리가 실제로 도움이 되지 않는다면 큰 다중 역할 체계를 만들지 않습니다.

가능하면 초기 구조는 다음 범위에 가깝게 유지합니다.

```text
.codex-plugin/
AGENTS.md
skills/
starter-prompts/
```

사용자의 워크플로에 분명한 필요가 있을 때만 더 추가합니다.

## 역할 선택 기준

초기 역할 수를 정할 때는 다음 편향을 사용합니다.

- `1개 역할`: 단순하거나 선형적인 작업
- `2개 역할`: 생성 후 검토가 필요한 작업
- `3개 역할`: 계획, 실행, 검토가 모두 필요한 작업
- `4개 이상`: 명확한 다단계 운영 필요가 있을 때만

## 출력 방식

긴 설명보다 구체적인 시작 파일을 우선합니다.

사용자가 빈 디렉토리에서 시작한다면 다음을 제공합니다.

1. 추천 폴더 구조
2. 첫 `skills/<role>/SKILL.md`
3. 바로 쓸 수 있는 스타터 프롬프트 하나

## 하지 않을 일

첫 결과를 문서 중심의 무거운 프레임워크로 만들지 않습니다.

사용자가 명시적으로 원하지 않는 한 깊은 reference 트리를 만들지 않습니다.
